**splitwm** is a standalone X11 tiling window manager in Rust that replicates a terminal-multiplexer-style split layout.

The architecture separates pure layout math (`tree.rs`, `state.rs`) from X11 event handling (`wm/`) and software rendering (`render.rs`). An arena-keyed n-ary tree (`NodeId` → `HashMap<NodeId, Node>`) keeps splits n-ary so collapsing a leaf redistributes its ratio proportionally to siblings; `theme.rs` is the single source of truth for all palette and metric constants, while `state.rs` owns every tree mutation (split, close, resize, focus, move-client-to-leaf, swap-leaves, toggle-minimize, insert-at-root, boundary resize, scroll, canvas width) so it never touches X11 directly. The `wm/` directory is a multi-file module: `wm/mod.rs` owns the event loop and arrange/compose pipeline; `wm/types.rs` holds the `Wm` struct, action enums, key definitions, and animation helpers; `wm/widgets.rs` computes hit-regions for drag handles, per-leaf "+" insert buttons, split-control buttons (close/split/swap/minimize, right-clickable to toggle direction), tab drags, edge insert buttons, and the bottom taskbar; `wm/menu.rs` implements open/close/hover/click logic for two override-redirect popup windows (main column + one category submenu); `wm/smush.rs` synthesizes Ctrl+0/Ctrl+- keystrokes via XTEST to auto-shrink terminal fonts in narrow splits, cached per client and width bucket. A cascading application launcher (`menu.rs`) scans freedesktop `.desktop` entries into categorized `MenuTree` data; per-leaf "+" buttons anchor the launcher below the leaf, and root-level "+" buttons at inter-column boundaries insert new empty columns. Tabs can be drag-and-dropped between leaves via pointer press/release tracking. Layout-changing actions trigger a wall-clock-driven ease-out-back animation (~280 ms) that interpolates frame rects with pre-split positions carried forward. A single full-screen underlay window composites the optional `SPLITWM_WALLPAPER` PNG, all leaf chrome, gap drag handles, insert buttons, the bottom taskbar, and the launcher menu via chunked `PutImage`, avoiding any compositing-manager dependency; `arrange()` dynamically grows the canvas width from leaf count and minimum split width, positions/maps off-screen clients out, and resamples the focused client's top-center pixel to adopt its accent colour (skipped mid-drag). Consecutive motion events within a batch are coalesced to the most recent pointer position so drag recomposites never pile up, and the full-screen BGRX blit buffer is reused across frames via a `RefCell<Vec<u8>>` to avoid per-frame allocation.
