# overview.md

splitwm is a from-scratch X11 tiling window manager written in Rust, implementing a terminal-multiplexer-style split layout on a scrollable canvas.

The architecture separates pure layout logic (`tree.rs`, `state.rs`) from X11 plumbing (`wm/`) and rendering (`render.rs`), with no reparenting — clients stay root children and all decoration is composited onto a single full-screen underlay window via software `PutImage` blits through MIT-SHM shared memory. The split tree is an arena-keyed n-ary structure (not boxed pointers), where branches enforce a two-or-more-children invariant so sibling lookups and ratio redistribution never need degenerate-branch guards. Same-direction nesting is forbidden at insertion and flattened on collapse, keeping every root-level boundary visible with "+" insert buttons.

All chrome is palette-indexed pixel art baked into the binary at compile time by `build.rs` (sprites are quantized to a fixed 16-colour palette at build, not decoded at startup), and palette-swapped per split via precomputed shade tables — the renderer never does runtime colour math for borders. App icons go through a one-shot OKLCH hue-rotation pipeline (not per-frame) to disambiguate same-app windows. The event loop is single-threaded with frame-paced rendering (~16ms vsync cadence), coalescing high-rate scroll and drag motion into one recomposite per frame; a separate D-Bus thread (with signals masked to the main thread) handles the notification daemon, waking the event loop via root `ClientMessage`.

A critical design invariant: `State` fields like `focused_leaf`, `scroll_x`, and `taskbar` are all private — every mutation routes through methods that enforce invariants (dangling focus can never be handed out, taskbar stays deduplicated, minimized leaves are never mapped). The panic guard in `wm::run` guarantees hidden windows are remapped on every exit path (clean, error, or panic) so no client is stranded invisible for the next WM. `WmError` classifies at construction time (fatal connection death vs. containable per-request failure), letting the event loop decide exit vs. recovery with a plain `match` instead of walking error source chains.
