**splitwm** is a standalone X11 tiling window manager in Rust that replicates the terminal-multiplexer-style split layout of a Lua-based AwesomeWM config, where persistent n-ary split trees hold tabbed leaf stacks on a horizontally-scrollable canvas.

The architecture separates pure layout math (`tree.rs`, `state.rs`) from X11 event handling (`wm.rs`) and software rendering (`render.rs`) — each leaf gets its own frame window with a custom slanted tab bar and rounded focus border composited via tiny-skia and blitted with `PutImage`, avoiding compositing manager dependencies. An arena-keyed tree (`NodeId` → `HashMap`) avoids Rust's borrow-checker friction for parent-child mutations, and the split tree is kept n-ary (not binary) so that collapsing a leaf reflows its ratio proportionally to siblings rather than forcing a binary rebalance. `theme.rs` is the single source of truth for all palette and metric constants; `state.rs` owns every mutation (split, close, resize, tab cycle, scroll, canvas width) so the event loop in `wm.rs` never touches the tree directly. Frame windows are positioned/hidden by `arrange()` which dynamically computes canvas width from leaf count and minimum split width, recomputes geometry on every mutation, maps off-screen frames out, and reparents active clients into the correct frame via a tracked `parent_leaf` field — a deliberate simplicity trade-off over per-client state tracking for reparent/visibility. Focus-direction navigation uses pure `usize` modular arithmetic throughout, avoiding signed-integer casts.
