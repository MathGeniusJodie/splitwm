**splitwm** is a standalone X11 tiling window manager in Rust that replicates the terminal-multiplexer-style split layout of the Lua-based "splitwm" AwesomeWM config, where persistent n-ary split trees hold tabbed leaf stacks on a horizontally-scrollable canvas.

The architecture separates pure layout math (`tree.rs`, `state.rs`) from X11 event handling (`wm.rs`) and software rendering (`render.rs`) — each leaf gets its own frame window with a custom slanted tab bar and rounded focus border composited via tiny-skia and blitted with `PutImage`, avoiding compositing manager dependencies. An arena-keyed tree (`NodeId` → `HashMap`) avoids Rust's borrow-checker friction for parent-child mutations, and the split tree is kept n-ary (not binary) so that collapsing a leaf reflows its ratio proportionally to siblings rather than forcing a binary rebalance. The `theme.rs` module is the single source of truth for all palette and metric constants ported from the Lua originals, keeping visual tweaks isolated; `state.rs` owns every mutation (split, close, resize, tab cycle, scroll) so that the event loop in `wm.rs` never touches the tree directly. Frame windows are positioned/hidden by `arrange()` which recomputes geometry on every mutation, maps off-screen frames out, and reparents active clients into the correct frame — a deliberate simplicity trade-off over tracking per-client state for reparent/visibility.
