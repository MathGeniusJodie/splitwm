**splitwm** is a standalone X11 tiling window manager in Rust that replicates the terminal-multiplexer-style split layout of a Lua-based AwesomeWM config, featuring animated transitions, gap drag-resize handles, insert-column buttons, auto font-shrinking (smush), app icons, content-sampled accent colours, and optional PNG wallpaper underlay.

The architecture separates pure layout math (`tree.rs`, `state.rs`) from X11 event handling (`wm.rs`) and software rendering (`render.rs`). Each leaf gets its own frame window with a custom slanted tab bar and rounded focus border composited via tiny-skia and blitted with chunked `PutImage`, avoiding compositing-manager dependencies. An arena-keyed tree (`NodeId` → `HashMap`) avoids Rust's borrow-checker friction for parent-child mutations, and the split tree is kept n-ary (not binary) so that collapsing a leaf reflows its ratio proportionally to siblings rather than forcing a binary rebalance. `theme.rs` is the single source of truth for all palette and metric constants; `state.rs` owns every mutation (split, close, resize, tab cycle, scroll, canvas width, insert-at-root, boundary resize) so the event loop in `wm.rs` never touches the tree directly. Layout-changing actions trigger an ease-out-back animation that interpolates frame rects over ~17 frames, with freshly created leaves starting from a collapsed sliver. Frame windows are positioned/hidden by `arrange()` which dynamically computes canvas width from leaf count and minimum split width, recomputes geometry on every mutation, maps off-screen frames out, and reparents active clients into the correct frame via a tracked `parent_leaf` field. Gap boundaries expose draggable handles and "+" insert buttons whose positions are updated each arrange pass, while a `smush` system synthesizes Ctrl+0/Ctrl+- keystrokes via XTEST to auto-shrink terminal fonts in narrow splits. Focus-direction navigation uses `usize` modular arithmetic on `collect_leaves()` ordering, and scroll state is kept on a horizontally-scrollable canvas wider than the viewport when needed.
