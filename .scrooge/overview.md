**splitwm** is a standalone X11 tiling window manager in Rust that replicates a terminal-multiplexer-style split layout with bitmap 9-slice decorations, a bottom taskbar, and a cascading app launcher.

Pure layout math (`tree.rs`, `state.rs`) never touches X11; `state.rs` owns every tree mutation (split, close, resize, focus, move-tab, toggle-minimize, insert-at-root, boundary resize, scroll, canvas width). An arena-keyed n-ary tree (`NodeId` → `HashMap<NodeId, Node>`) keeps splits n-ary so collapsing a leaf redistributes its ratio proportionally to siblings; `theme.rs` is the single source of truth for all palette and metric constants, including a 16-colour `na16` palette with per-leaf accent indices cycled from a curated subset. The `wm/` multi-file module owns the X11 event loop (`wm/mod.rs`), the `Wm` struct and action enums (`wm/types.rs`), hit-region computation for gap drag handles, per-leaf bitmap split-control buttons (close/minimize/split with dedicated disabled-variant PNGs), tab drag regions, edge insert buttons, and the bottom taskbar (`wm/widgets.rs`), and open/close/hover/click logic for two override-redirect launcher popup windows (`wm/menu.rs`). A cascading application launcher (`menu.rs`) scans freedesktop `.desktop` entries into categorized `MenuTree` data; per-leaf "+" buttons at inter-column boundaries and edge insert buttons add new empty columns, and a taskbar "+" opens the launcher. `render.rs` builds a `Renderer` on tiny-skia that composites all leaf chrome — bitmap 9-slice window borders (`NineSlice`) palette-swapped per accent via `pixel-graphics`, minimized restore strips (`MinimizedSlice`), embedded close/minimize/split PNG buttons with disabled variants, per-leaf app icons, gap drag handles, insert buttons, the taskbar, and the launcher menu — onto a single full-screen underlay pixmap, converts to BGRX, and pushes via chunked `PutImage`, avoiding any compositing-manager dependency. Client icons are extracted from `_NET_WM_ICON` properties and rendered into titlebars and taskbar tiles; a persistent hue-rotated accent colour per leaf (`leaf_color` by node ID) colours the focus border and taskbar highlight. Layout-changing actions trigger a wall-clock-driven ease-out-back animation (~280 ms) that interpolates frame rects with pre-split positions carried forward. `arrange()` dynamically grows the canvas width from leaf count and minimum split width, positions/maps off-screen clients out, and the full-screen BGRX blit buffer is reused across frames via a `RefCell<Vec<u8>>` to avoid per-frame allocation. Consecutive motion events within a batch are coalesced to the most recent pointer position so drag recomposites never pile up.
